Store compliance — exact console click-paths

The two console gates for launch: Apple App Privacy labels and Google Play Data Safety. The audited values live in compliance-declarations.html; this doc is the screen-by-screen sequence to enter them. Open this on a second screen and click along.

One fact drives every answer: the app collects exactly 4 data types — email, name, user ID, message content — all collected, never shared with third parties, never used for tracking, all linked to the user, all encrypted in transit (HTTPS/WSS), with in-app account deletion. That's it. No analytics, no ads, no crash SDK, no location/contacts/photos. If a screen offers a type not in that list of four, the answer is leave it off.

1. Apple — App Privacy App Store Connect

→ Open App Privacy (Aiko Chat, id 6785395468)

If that deep-link doesn't land on the page: appstoreconnect.apple.com → Apps → Aiko Chat → left sidebar App Privacy → Get Started (first time) or Edit (next to "Data Types").

  1. Data collection question: "Do you or your third-party partners collect data from this app?" → Yes, we collect data from this app.
  2. Select data types. Tick exactly these four; leave every other box unticked:
    Apple category → typeTick
    Contact Info → Email Address✓
    Contact Info → Name✓
    Identifiers → User ID✓
    User Content → Other User Content  (this is the messages bucket)✓
    Click Save → Next.
  3. For EACH of the four types, Apple asks three questions. Same answers every time:
    QuestionAnswer (all four types)
    How is this data used? (purposes)App Functionality — and nothing else
    Is this data linked to the user's identity?Yes
    Is this data used for tracking?No
  4. "Data Used to Track You" and "Data Not Linked to You" sections → None. Add nothing.
  5. Privacy Policy URL (same App Privacy page, or App Information): set to https://imagineering.cc/aiko/privacy/ (live, verified 200).
  6. Publish. Apple's labels publish with the next app version / can be published from this page. Click Publish when prompted.
The "Other User Content" type is the one people miss — messages are User Content, not an "identifier". Purpose is App Functionality (delivery + history), never "Analytics" or "Product Personalization".

2. Google — Data Safety Play Console

→ Open Play Console

Deep-link needs your numeric developer + app IDs (I don't have them), so this lands on the console home. Navigate: select Aiko Chat (cc.imagineering.aiko_chat_app) → left nav Policy and programs → App content → find Data safety → Start (or Manage to edit).

Play's Data Safety is a multi-screen wizard. In order:

Screen — Overview

  1. Read, click Next.

Screen — Data collection and security

QuestionAnswer

"Shared" in Play's sense = transferred to a third party. The gateway is your own first-party server, and the sign-in providers act on the user's instruction to authenticate — so No sharing is correct for all four. Do not tick any App activity / App info-and-performance type (no analytics or crash SDK exists).

Screen — Review & submit

  1. Review the summary; it should list exactly the four types above, all "Collected, not shared".
  2. Also set the Privacy Policy URL (Store presence → Main store listing, or the Data safety form if it asks): https://imagineering.cc/aiko/privacy/.
  3. Submit.
