A five-scene explainer

Two tongues, one sea

Aboard, your ship keeps a rich private language for everything it carries. But what crosses the water has to travel sealed in a plain, shared tongue — and something at the dock has to translate between the two, faithfully, every time.

Scroll to explore
Scene 01

The ship’s own tongue

The idea

Everything the app does — drawing a bubble, greying out a pending send, sorting the timeline — it does against neat little shapes it keeps in its own hold. A message aboard isn’t raw text; it’s a tidy record with a name, a sender, a body, and a delivery state the ship can reason about. This is the ship’s private language, and the whole rest of the app is written to speak it.

The domain models: immutable Dart shapes the app is written against — a Message with a MessageSender, a MessageKind, and a DeliveryState (sending · sent · failed). Rich, hand-written, and never seen by anyone off the ship. Nothing here knows anything about JSON or the wire.
Scene 02

Sealed for the crossing

The idea

The moment a message leaves the ship, none of that private richness can go with it. Anything that travels the water has to be written in a flat, common tongue every port already agrees on — so it rolls the note up, seals it in a bottle, and casts it off. The bottle is deliberately plain: a handful of labelled fields, nothing the ship couldn’t reconstruct at the far shore.

The wire envelopes: thin JSON DTOs in one file, envelopes.dart — a SendFrame going out ({"type":"send", "client_msg_id", "body", …}) and a MessageView coming back. Note the outbound bottle carries no sender field: the far shore stamps who you are from your token, so the ship never writes it.
Scene 03

The translator at the dock

The idea

Between the ship’s rich tongue and the bottle’s plain one stands a single translator, working only at the dock. Everything outbound gets folded down into a bottle here; everything inbound gets unrolled back into a ship’s record here. Keeping this one person at this one spot means the ship never learns a word of the common tongue, and the sea never sees a word of the ship’s — and when the shared language changes, only the dock has to relearn it.

The mapping: conversion functions between domain and wire — MessageView.fromJson unrolls an inbound bottle into a Message, ServerFrame.fromJson sorts each arrival into a sealed frame type. The join that ties the two worlds together: client_msg_id == clientTempId, the same value on both sides. All wire knowledge lives in this one file.
Scene 04

The name that survives the voyage

The idea

Here’s the clever part. The instant you hit send, the ship ties its own permanent tag to the message — a name it minted itself, before the bottle ever touched water. When the far shore receives it, it sends back a receipt bearing an official harbour number and the true time. The ship finds its message by the tag it never let go of, presses the official number on beside it, and flips the state to sent. The tag was the anchor the whole time.

The two-id design: clientTempId is the eternal key, minted on the ship and durable forever; id (the server ULID) stays null until the ack arrives, then reconciles against the row by matching clientTempId. Because id is truly nullable end-to-end, an un-acked message survives a save-and-reload without a single throw — load and write stay symmetric.
Scene 05

A bottle from a newer ship

The idea

One day a bottle washes in flying a flag the dock has never seen — a message kind, or a whole frame type, added by a newer, updated fleet. The translator doesn’t panic and it doesn’t sink the boat. It files the unfamiliar flag under “some external voice”, keeps the message moving, and logs the oddity for later. The one thing it won’t abide is a bottle that’s genuinely malformed — that gets stopped right here at the seam, before it can confuse the ship.

Validation at the seam, leniently: an unknown SenderKind decodes to actor, an unknown frame becomes an UnknownFrame (logged, dropped, never thrown), and an inbound "label": null passes clean. Forward-compat is the rule — a newer server must never crash an older client — while the mapping boundary stays the one place a truly broken payload is caught.

The whole thing, in one breath

Rich aboard, plain at sea, faithful at the dock.

The app thinks in its own comfortable shapes, the water only carries plain sealed bottles, and one careful translator moves between them — minting a name that survives the crossing, and staying open to bottles from ships it hasn’t met yet without ever letting a broken one aboard.

The trick that makes it all safe is boring on purpose: keep the two languages apart, and put one door between them. Every wire change touches only the dock; every domain change never leaves the ship. The seam is where trust is decided — a plain envelope delivered is not yet a message believed, so it’s parsed, checked, and only then let aboard.
